### Features
- **CC automation now records as latch overwrite.** Recording a knob on the AUTOMATION (AUTO) bank no longer just punches in points while you turn — it now works like DAW *latch* automation. The instant you **turn** a knob (record-armed and playing) it latches and from then on continuously **overwrites** that lane along the playhead with the knob's current value, replacing whatever automation was there — and keeps writing the last value even after you stop turning, loop after loop, until you stop recording. Turn it to a value and leave it → the lane flattens to that value; keep nudging → you draw the curve live; the first turn starts from the value already playing there, so there's no jump. **Touching** a knob without turning now does nothing (the old write-while-held behavior is gone). Untouched knobs keep playing their existing automation. After each loop (and when recording stops) the lane is **decimated** — redundant points on a flat hold or straight ramp collapse to their endpoints, losslessly — so long recordings stay clean and never run out of points. (Also fixes a latent overflow: the per-lane point counter was 8-bit while the cap is 1024.)
- **Melodic pad pressure → aftertouch, live + recorded.** New per-track **Track Config → AftTch** setting forwards pad pressure (poly aftertouch) from melodic pads to the track's output as you play: hold a note and press harder for expressive control. Modes are **Off / Poly / Channel** — on **Move**-routed tracks only **Off / Poly** (Move instruments take poly aftertouch); **Schwung** and **External** tracks add **Channel** (one track-wide channel-pressure value). Default **Off**. Poly mode carries each pad's own sounded pitch. Routed through the normal output path so it reaches the Move synth, the Schwung chain, or USB-A exactly like notes. **Pressure is recorded into the clip** when the track is record-armed (monitored but not captured during count-in) and **plays back** as interpolated automation on every loop — independent of the AftTch toggle (the toggle gates *incoming* AT; recorded AT always plays until cleared). The toggle is hidden on drum tracks (there pad pressure drives repeat velocity).
- **CC PARAM bank is now the AUTOMATION bank ("AUTO").** Renamed and generalized to host all per-clip continuous-modulation data. The header shows inverted **AT** / **CC** badges for whichever automation types the focused clip contains (Pitch Bend slot reserved for later). **Tap Delete** (in the bank, without turning the jog) opens a **CLEAR AUTOMATION** menu — jog to scroll, jog-click to check **AT** and/or **CC**, then **CLEAR** (Note/Session cancels); Delete+knob and Delete+step still do their per-knob / per-step CC clears. **Delete+jog** and **Shift+Delete+jog** now clear **all** automation (CC + AT) for the clip, and **clearing a clip** also removes all its automation.
- **CC PARAM bank reworked into a per-clip continuous-modulation editor.** Each melodic **clip** now owns its 8 CC lanes: recorded automation (cap raised 64 → **1024 points/lane**, so a long continuous sweep records across the whole clip instead of dropping after ~2 bars) plus a per-clip **resting value** ("clip CC"). The resting value is **opt-in** — every knob's range gains a **"—" floor** (turn down past 0 → "—" = send nothing; turn up from "—" → 0…127). With a resting value set, the lane eases back to it at every loop boundary (a seamless closed curve that resets each cycle); left at "—" the value carries over the loop (today's hold behavior). Knob-turn rules: **stopped** (or playing on an un-automated lane) sets the clip resting value live; **record-armed** records automation; **playing + automated** auditions transiently without touching the resting value. **Held step** shows the value recorded there (or "—") plus the **computed value that plays there** in parens for ramp/gap steps; step-edit writes a clean flat hold. The bank's **OLED now stays up** the whole time you're on it. The **last-touched knob is the active lane** (its overview cell stays highlighted) and its automation is drawn across the **step LEDs as a 3-level white brightness gradient** ("—"=off, dim/mid/full), with the **playhead step in the track color**. The knob ring LEDs show solid status colors: **red** = recording, **green** = automation playing back, **yellow** = automation exists, **white** = resting value set, off = empty. Single-step clears: hold a step + turn a knob **down past 0** clears that knob's point there; **Delete+step** clears all knobs at that step. Resets (Delete+turn per knob, Delete+jog, Clear Session, hard-reset clip) clear automation **and** the resting value back to "—". Each knob also gains a **type** (Shift+turn ladder: **AT** = channel-pressure aftertouch ↔ CC0…CC127); aftertouch sends `0xD0`. Per-clip resting values, per-knob types, and the larger automation persist with the set (text state, no version bump). Knob→CC **assignment stays per-track**.
- **CC bank knobs gain acceleration.** Editing a CC value (resting value, recording, or step-edit) now ramps: the first few detents of a continuous turn move finely (≈1 value per 3 detents), and sustained turning speeds up so big sweeps are quick. A direction change or a pause resets to fine. Run-length based (the Move knobs send fixed ±1 detents regardless of speed, so timing can't tell slow from fast).
- **Save states (snapshots) — multiple save/load per set.** The Global Menu's single `Save` is replaced by **Save state** + **Load state**. **Save state** writes a timestamped snapshot (`MM-DD HH:MM`) of the full project — clips, notes, track config, tempo, scale — up to **16 per set**; **Load state** opens a jog-scrollable list (newest first), and picking one (Yes/No confirm, since loading discards unsaved changes) restores it via the normal set-reload path. At the 16-snapshot cap, **Save state** opens an overwrite picker to choose which to replace (with confirm). Snapshots are per-set, stored as flat `seq8-snap-*` files beside the live state, tracked by a `seq8-snap-index.json` manifest (no directory enumeration). **Clear Session keeps snapshots** (they're independent recovery points). After a future state-format bump, incompatible snapshots are marked `(old)` and the Load list offers to remove them on first open. JS-only — no DSP change; save reuses the deferred `save` flush then copies the on-disk state into the snapshot one tick later, load reuses `pendingSetLoad`. New `STATE_VERSION` constant mirrors the DSP `v=32`.
- **Export to Ableton Live (.ablbundle).** *(Phase 1 — session skeleton.)* New **Global Menu → Export to Ableton** action writes the current set as an Ableton `.ablbundle` that desktop Live opens directly (then *Save As* `.als`). Export requires a **stopped transport** (otherwise a 1-second **STOP TRANSPORT / FOR EXPORT** notice and no-op) and is gated behind a Yes/No confirm dialog (No default; jog to choose, Back to cancel). Output lands at `/data/UserData/schwung/davebox-exports/<set name>-<date>.ablbundle` (same-day repeats get `-2`, `-3`, …), retrievable over SFTP. This first cut produces the **8 MIDI tracks × 16 scene slots** structure carrying tempo and key, every track seeded with a placeholder Drift instrument so Live accepts the file; per-track instruments/samples and baked clip notes arrive in later phases. Fully on-device and offline — JS authors the `Song.abl`, a bundled `pack.py` (invoked via `host_system_cmd` → stock `python3`) copies samples and builds the store-mode ZIP. No patched-Schwung dependency.
- **Export to Ableton: multi-cycle bake for randomized/delayed clips + bake-fidelity fixes.** *(Phase 4b.)* Randomized melodic clips now export **8 cycles** of distinct variations (instead of one frozen loop), and clips with **Delay** are wrapped so echoes loop seamlessly; the loop brace spans the whole clip. "Randomized" covers NOTE FX Pitch Random, the Rnd/RnO arp styles, and the Delay bank's feedback Pitch Random. Alongside this, the bake (which also powers the in-app **Capture/bake**) was audited and corrected to match live playback faithfully: it now honors **Delay Retrigger** (echoes are cut when a new note retriggers, instead of piling up), uses the correct **echo note length** (echoes were baking far too long — sustained instead of the staccato you hear), and applies the **Arp Steps** per-step interval offsets and loop length (previously ignored, and the step loop was stuck at 8). Drum delay retrigger is honored too.
- **Export to Ableton: progress + confirmation polish.** *(Phase 6.)* The OLED now shows **EXPORTING…** while the packager runs (instead of looking frozen), then a **persistent "EXPORTED TO" screen with the full file path** that stays up until you dismiss it with OK (jog-click or Back) — so you can read where the bundle landed. The scratch workspace now lives under `davebox-exports/staging` (keeping the schwung folder uncluttered) and is reliably deleted after each export.
- **Export to Ableton: self-contained samples.** *(Phase 5.)* Exported bundles now embed the actual instrument samples, so an export opens with all its sounds even on a computer without the Move sample packs installed. Each instrument's sample references are resolved to their files (CoreLibrary / User Library), copied into the bundle's `Samples/` folder (deduplicated), and rewritten as relative references. Samples are copied byte-for-byte by the on-device packager. If a referenced sample can't be found, the export still completes and the OLED notes how many were missing.
- **Export to Ableton: drum clips (polymeter flatten).** *(Phase 4.)* Drum clips now export too. Because dAVEBOx drum lanes can each loop at a different length (polymeter), a single Ableton clip can't hold per-lane loops — so the export renders each lane's baked hits and tiles them onto one clip whose length is the **least common multiple** of the active lanes' loop lengths (computed in ticks). The result loops seamlessly in Live with every drum keeping its own cycle. Each lane's notes land at its drum-cell pitch; same-lane overlaps are legalized like melodic clips. Coprime lane lengths are capped (64-bar / 2048-note ceiling, snapped to a clean multiple of the longest lane). DSP-side render via `tN_cC_export_drum`.
- **Export to Ableton: baked melodic clip notes.** *(Phase 3.)* Each non-empty melodic clip now exports its **baked "what you hear"** notes — the chain effects (SEQUENCE ARP, MIDI Delay, Harmony, gate length) are rendered into real MIDI so the exported clip plays in Live exactly as it sounds in dAVEBOx, no live FX required. Implemented as a DSP non-destructive render (mirrors the bake compute but emits via `get_param` without mutating the clip), read per-clip in JS and converted ticks→beats. Same-pitch overlaps that baked effects can produce (legal as live MIDI but rejected by Ableton clips — "Document invariant violation") are automatically legalized: each note is clamped to end just before the next same-pitch onset. Drum clips and the multi-cycle loop-brace layout for randomized/delayed clips come in later phases.
- **Export to Ableton: route-aware instruments + names.** *(Phase 2.)* Exported tracks now carry the right instrument and name per route: **Move-routed** tracks get the actual Move instrument for their channel (drum kit or melodic, matched via the loaded set's `midiInputMode`), named after the Move preset and using its track color; **Schwung-routed** tracks are named `SCH-<chain name>` (from `shadow_chain_config.json`); **External** tracks are named `Ext ch <n>`; anything unmatched falls back to a placeholder Drift named `dB <n>`. (Move instruments are still referenced from installed Ableton packs — self-contained sample bundling is a later phase.)
- **Schwung chain-edit co-run: knobs now drive chain params.** *(Patched-Schwung only — capability-gated.)* While in **Edit Slot…** co-run, the knob row (K1–K8) controls the focused chain component's parameters — turn to adjust, touch to peek the current value — with the slot-global knob-macro fallback at the chain top level, mirroring the native chain editor. Routing is wrapped to the editor's view (the editor's knob-context cache is view-keyed) and falls through to dAVEBOx so knob-touch state stays balanced on exit. The value popup's ~4 s lifetime is unchanged native behavior. Requires the `legsmechanical/schwung` shim.
- **Move-native co-run: reliable track landing + clean LED handoff.** *(Patched-Schwung only — capability-gated; invisible on stock Schwung.)* Entering **Edit Synth…** co-run now reliably lands Move on the track the dAVEBOx track is routed to — previously intermittent (you'd open co-run looking at the wrong Move track and have to hunt for the right one). Implemented as an alternating neighbor→co-run track-button inject sequence that forces a definitive selection. LED ownership in co-run is now clean: the shim passes Move's **knob-ring + master** LEDs through to hardware while dAVEBOx keeps **pads / step buttons / transport**; dAVEBOx blinks the **4 track buttons dark-grey** as a "press to switch Move track" affordance (we dropped chasing Move's native track colors — the firmware only repaints a single track on selection, so a full native row was never reliable). On **entry** the knob rings light without a manual touch; on **exit** dAVEBOx immediately reclaims the knob-ring LEDs (previously Move's colors lingered until you turned a knob). Requires the `legsmechanical/schwung` shim (co-run selective LED filter + the upstream May-22 LED snapshot/restore commits).
- **Track type conversion carries notes over (Track Config → Mode).** Flipping a track between **Keys** and **Drums** now translates the sequenced notes so the music follows the track, instead of just swapping the view. Notes only — time/pitch/velocity/gate are preserved; effects, arp, and harmony reset to defaults (not carried). **Drums → Keys**: every lane's hits become pitched notes at the same times, simultaneous hits become chords (converts immediately). **Keys → Drums**: each distinct pitch maps to its own drum lane (lowest pitch on the first pad, ascending); a part using more than 32 distinct pitches keeps the 32 most-used; a confirm dialog (`Warning: / Existing notes may be lost. / Proceed?`, No default) precedes it. An **empty track converts instantly with no dialog** and skips the per-clip work (DSP `track_is_empty` early-out; JS skips the dialog + the all-track resync). New per-track DSP keys `tN_convert_to_drum` / `tN_convert_to_melodic` translate the note data and flip `pad_mode` atomically in one set_param. State format unchanged (v=32).
- **ARP IN bank reset** via Delete+jog while bank 5 (ARP IN) is active. Resets *all* TARP params — style, rate, octaves, gate, steps mode, retrigger, latch, sync, step velocities, step intervals, step-loop length — in a single gesture. New DSP key `tN_tarp_reset` resolves to `arp_silence` + `tarp_drop_latched` + `arp_init_defaults(&tr->tarp)` + held-buffer + tarp_on/latch/sync defaults. JS-side `resetTarp(t)` zeroes the bank-5 mirrors in parallel. Shift+Delete+jog (the broad melodic FX reset) intentionally still leaves ARP IN untouched — only Delete+jog while ARP IN is the active bank resets it.
- **Arp Steps overlay gates pad-note dispatch.** While the SEQ ARP / ARP IN step-edit overlay is active, pads no longer trigger notes when tapped — they're purely the persistent vel-level editor and the Loop-held loop-length picker. `_padDispatchMutedNow` now returns true for `stepIntervalMode && (bank === 4 || bank === 5)`; `computePadNoteMap` is re-pushed on every overlay enter/exit.
- **Note/Session button exits the Arp Steps overlay** while in Track View, without switching to Session View. One press out, no need to jog the wheel.
- **Loop button is repurposed as a modifier inside the Arp Steps overlay.** While the overlay is active, Loop press/release no longer touches TARP latch, drum repeat latch, or the loop-window gesture — it only sets `S.loopHeld` so the pad-press handler can read it for the loop-length gesture.
- **Arp Steps step pattern loop length.** Inside the Arp Steps overlay, **hold Loop + tap any pad on a step column** to set the pattern's loop length (1..8). Loop length governs both `step_vel` and `step_int` — the arp cycles through `step_pos = 0..L-1` instead of the fixed 0..7. Per-clip for SEQ ARP, per-track for TARP. Default = 8. Pads in columns past the loop length render off in the persistent pad editor (length is visible from the pad-grid alone — no OLED indicator). New DSP fields `arp_engine_t.step_loop_len` + `clip_pfx_params_t.seq_arp_step_loop_len`; new keys `tN_seq_arp_step_loop_len` + `tN_tarp_step_loop_len`; sparse persistence as `t%dc%d_arsll` / `t%d_tasll`. State version bumped v=31 → v=32.
- **Arp Steps bank header renamed per source.** SEQ ARP overlay header reads `SEQ ARP Steps`; TARP overlay reads `ARP IN Steps`. Previously both showed the generic `Arp Steps`.
- **Arp Steps interval-edit bank.** While on SEQ ARP (bank 4, melodic per-clip) or TARP (bank 5, per-track), **click the jog wheel** to enter a persistent step-interval overlay. K1..K8 set per-step scale-degree offsets (±14, scale-aware via existing `scale_transpose`) on top of whatever pitch the arp picks for that step. Bank heading reads `Arp Steps`; knob cells show `S1..S8` with the signed value. Pad grid stays in the familiar 4×8 step-vel level editor (same view that used to require K5-touch) — persistent while the overlay is active, so you can edit both vel-level and step-interval without juggling knob touch. Jog turn or jog click exits. The old **K5-touch → step-pad gesture is removed** on these two banks (replaced by the persistent jog-click mode). Per-clip persistence for SEQ ARP, per-track for TARP. State version bumped v=30 → v=31 (existing dAVEBOx state files are wiped on first launch — pre-public).

### Fixes
- **Track-bank OLED returns to the overview faster.** After selecting a bank, the OLED held the bank's parameter view for ~4.2 seconds before falling back to the track overview — the timeout constant was calibrated for an old 196 Hz tick rate but the device runs at ~94 Hz, so it lingered about twice as long as intended. It now falls back after ~1 second.
- **Recording CC automation no longer fights the knob.** On the AUTOMATION (AUTO) bank, recording a CC value by turning a knob barely moved and kept snapping back toward zero. The realtime display value the UI polls (`cc_cur_vals`) doubles as the running base the next knob detent accumulates from; during recording it was being captured from the *playhead's* automation output, which — just after you write a point, with no resting value set — is the lane's "—" tail. So every poll between detents reset the base to nothing and the value never climbed. While a knob is actively being recorded it now reports the live value being written instead, so successive detents accumulate normally.
- **Drum repeats now respond to pad pressure.** Holding a drum repeat pad (Rpt1 or Rpt2) and pressing harder sets the velocity of the incoming repeats — the feature was coded but never fired because incoming pad aftertouch was being dropped as "noise" on the first line of the MIDI handler. Aftertouch is now handled before that filter.
- **Holding a recorded CC step now shows the value recorded there**, not the live knob value. Previously the step editor seeded all knobs from the current live CC value, so a held step displayed whatever the knob last sent rather than the automation point at that step. The step editor now reads the recorded point per knob (showing "—" where no point exists) and a single owner (`_onCC_stepedit`) writes step edits — the duplicate write path in the knob handler was removed. (This was the seed of the broader CC PARAM bank rework above.)
- **Orphan-state prune now removes snapshot files too.** When a Move set is deleted, dAVEBOx's launch-time cleanup (`prune_orphan_states`) removes that set's leftover state files. It previously only unlinked `seq8-state.json` + `seq8-ui-state.json`, so a deleted set that had save-state snapshots left its `seq8-snap-*` files (and the folder) behind. The prune now enumerates the orphaned set's folder and removes any `seq8-snap-*` files before the directory cleanup. Live sets are unaffected.
- **Note-duration step LEDs now match the played length.** When holding a step and tapping another to set a note's gate, the highlighted step LEDs lit one step too many (e.g. hold step 1, tap step 3 highlighted steps 1–4 instead of 1–3). The gate itself was correct; the LED overlay used `floor(gate/tps)` inclusive, which over-counts whenever the gate is an exact number of steps (every such gesture). Now lit steps = `ceil(gate/tps)` of actual coverage, so the highlight matches what the note plays (melodic and drum step edit).
- **Hot-path debug log probes gated behind a compile-time flag (`SEQ8_DEBUG_PROBES`, default off).** Several `seq8_ilog` debug probes were left compiled into shipped builds — most importantly a `Z4 INSERT` probe that fired a synchronous `fprintf`+`fflush` on *every* clip-note insert. Because step edits rebuild a clip's `notes[]` from its step arrays (via `clip_migrate_to_notes`, called from ~55 set_param handlers on the audio thread), a step-velocity knob sweep produced a storm of forced file writes on the real-time audio thread — enough to trip kernel RT throttling and freeze the device. All five hot-path probes (`Z4 INSERT`, both `repeat-invariant` checks, both `WINDOW SNAP` logs) now expand to nothing unless `SEQ8_DEBUG_PROBES=1`; the `WINDOW SNAP` playhead-snap correction itself stays live. Operational logs (state load, init/destroy, dropped-data warnings) are unchanged. No state-format change (v=32).
- **Volume knob no longer stutters sequencer playback.** Turning the master volume knob (CC 79) while a pattern played caused note stutter and MIDI-to-Move pauses. The host forwards every CC 79 detent to dAVEBOx in overtake mode (`claims_master_knob`), but volume is owned by Move firmware (`button_passthrough[79]` + Schwung's overtake volume-overlay passthrough), so dAVEBOx had nothing to do with the stream except spend cycles dispatching it against the sequencer/MIDI path. CC 79 and its capacitive-touch note (8) are now dropped at the top of `onMidiMessageInternal`. Volume adjustment stays fully Move-native; the 8 device knobs (CC 71-78) are unaffected. (Was masked in Move-native co-run, where the shim already filters CC 79 away from the tool — the hiccup only showed outside co-run.)
- **Shift+Step3 (Edit Slot / Edit Synth co-run) is now Track View only.** Previously also fired in Session View, where it isn't a documented shortcut. The Session-View shift-shortcut LEDs (icons CC 16-31 and step pads 16-31) no longer light step 3 — only the shared menu shortcuts (Global, Tap Tempo, Metronome, Swing, Scale) are surfaced there.

### Features
- **Bank header now shows a `Tr[n]` track indicator** on the right side of every parameter bank's heading. Updates live as the active track changes. Patched in both `drawBankHeading` / `drawBankHeadingInverted` and the ALL LANES inline header.
- **Bake moved to Capture button; Sample button is now Multi-Track Live Merge.** Bare-tap rebinding (held-as-modifier gestures unchanged). Track View: tap **Capture** opens the clip-bake confirm dialog; tap Sample is a no-op. Session View: tap **Capture** opens a **scene-bake picker** ("Tap row or scene step to pick destination") — picking a row routes to the scene-bake confirm dialog; any other input cancels. Tap **Sample** arms/stops a new multi-track live merge that captures all 8 tracks simultaneously into a deferred-placement buffer. On stop, a "PLACE MERGED CLIPS" prompt asks the user to tap a scene row; the captured clips land on that row. **Placement is selective per-track**: tracks that captured zero notes during the window leave their existing clip at the destination row untouched; tracks with captured data overwrite their column. Held-modifier gestures (Sample-held + scene row = scene bake; Capture-held + scene row = scene capture; Capture-held + drum pad = lane select) all work as before.
- **Scene bake now has a wrap-tails phase** mirroring clip bake (loop count → Wrap YES/NO). Was loop-count only before. Same `bake_scene` DSP key extended with a third token; defaults to no-wrap when omitted.
- **Live merge entry shows a 3-line OLED popup** ("LIVE MERGE / Capturing all 8 / tracks. Tap Sample / again to stop.") for ~3 seconds so the gesture isn't silent. The action-popup renderer was extended to 3- and 4-line layouts (Session View and bank-overview render sites both updated).
- **Multi-track merge captures the full chain output across all 8 tracks** with per-track pending buffers sized to `MAX_NOTES_PER_CLIP` (512) so even drum-heavy long captures don't hit a per-track cap. Capture continues during the `STOPPING` window (between tap-Sample-to-stop and the next bar boundary) so trailing notes in the final partial bar aren't dropped.
- **New DSP keys** `launch_scene_quant` (already shipped earlier on a different rev), `merge_arm` (now zero-arg — all tracks), `merge_place_row N` (commit captured pending to row N), `merge_cancel` (discard captures, return to IDLE). `merge_state` exposes a new value `4 = CAPTURED` (awaiting placement) used by the JS dialog gate. Single-track merge removed; `merge_track`/`merge_dst_clip` fields retired; `state_full` snapshot byte 56 (former `merge_dst_clip`) no longer emitted (JS guards on length, so older builds gracefully ignore the missing slot).
### Fixes
- **Transport start no longer auto-launches every track's focused clip.** The "focused clip plays by default" contract was over-applied: on Play, the auto-launch loop iterated all 8 tracks and queued each track's `trackActiveClip`. This silently undid Session-View deactivation gestures — Delete+Play to stop everything, or tap-clip-to-stop on individual tracks, would be undone by the next transport start. Spec is single-clip: only the clip currently open for editing in Track View auto-launches; Session-View Play only launches what was explicitly queued. Gated to `!S.sessionView` and the active-track-only single-clip launch. The track-switch auto-launch (`_switchActiveTrack`, already gated to Track View) is unchanged.
- **DELAY bank K7 (Retrg) value no longer "resets to OFF" on clip switch.** The DSP `pfx_snapshot` (read by `refreshPerClipBankParams` on every clip switch) was still emitting `fb_clock` at slot 15, but slot 15 maps to bank K6 in the JS bank-overview mirror — which after the K7=Retrg rebind should be `delay_retrig`. Every clip switch was therefore overwriting `S.bankParams[t][3][6]` with whatever clock_fb happened to be (usually 0), flipping the K7 display to OFF even though the DSP-side value was retained correctly. Fix: melodic snapshot slot 15 now emits `cp->delay_retrig`; drum-lane snapshot appends `dp->delay_retrig` as the 10th value and the JS reader maps it to `bankParams[t][3][6]`. Also defers the `tN_delay_retrig` `set_param` via `pendingDefaultSetParams` so a fast K7-then-clip-switch gesture can't coalesce-drop the write (same-track sync `tN_*` exception).
### Features
- **Delay Retrig knob on the DELAY bank (K7).** New 0/1 toggle: when ON (default), an incoming note-on drains in-flight delay echoes for that track before scheduling its own, so newer notes don't pile on top of stale repeat tails; when OFF, repeats overlap (the prior behavior). K7's prior role — Clock Feedback — folds onto **Shift + K1** with a dynamic OLED label flip Rate ↔ ClkF and the live clock-feedback value shown in K1's value cell while Shift is held. Drum tracks expose Retrig on K7 too (was previously blocked); drum K6 still maps directly to clock_fb via the existing per-lane remap, so no Shift+K1 alt is needed on drum. Each turn diffs the previous Bjorklund pattern against the new one: only the changed positions toggle, so hand-placed hits outside the Euclidean set are preserved across knob sweeps. Hits are written with the unified step-entry velocity. Existing knobs shift right one slot: Len → K5, SqFl → K6 (K7/K8 LaneOct/LaneNote unchanged). Value is persisted per-track-per-lane via UI sidecar `dleu`.
- **Capture + scene-row button** — snapshots the current performance into the pressed scene row: copies each track's currently-active clip into that row in one gesture. Skips tracks where the active clip is already on the target row (self-copy) or empty (so target keeps its prior contents on unused tracks). View-agnostic — works in both Session View and Track View. OLED shows `CAPTURED / TO ROW N` (or `NOTHING / TO CAPTURE` if nothing applied).
- Track menu: **Edit Slot...** — hands the OLED + jog wheel + track buttons over to Schwung's native chain editor while dAVEBOx keeps pads, step buttons, knobs, and transport. First use shows a slot picker (1-4); the choice persists per track. Hold **Shift** while selecting `Edit Slot...` to reopen the picker and reassign. Press **Back** to navigate up within the editor, **Menu** to exit co-run. Entry is shown only on **Schwung-routed tracks** (symmetric with Edit Synth, which is shown only on Move-routed tracks). **Capability-gated**: requires the patched Schwung shim from [`legsmechanical/schwung`](https://github.com/legsmechanical/schwung) (adds `shadow_set_corun_chain_edit` + `corun_chain_edit_slot` in shadow_control). On stock Schwung the entry is hidden and the feature is invisible.
- Track menu: **Edit Synth...** (Phase A) — for tracks set to **Route: Move**, hands the OLED + jog + track buttons + Shift + Back + device-edit knobs + master knob to Move firmware's native preset browser and device-edit pages, while dAVEBOx keeps pads, step buttons, transport, and Menu. The sequencer keeps firing audibly so you can audition presets and parameter changes against the playing pattern. Auto-taps the matching track button on entry based on the track's MIDI channel (channel 1-4 → Move tracks 1-4); on channels outside 1-4 the user picks the Move track manually via Move's track buttons. **Menu** exits co-run. On **Drum**-mode tracks: tapping a pad in the left 4 columns silently selects the corresponding cell in Move's drum-instrument editor (via a synthesized Shift+pad gesture) while dAVEBOx still fires the drum from its sequencer — no double trigger. **Capability-gated**: requires the patched Schwung shim from [`legsmechanical/schwung`](https://github.com/legsmechanical/schwung) (adds `shadow_set_corun_move_native` + `corun_move_native_track` in shadow_control). On stock Schwung or on non-Route-Move tracks the entry is hidden. Implementation note: pad and step-button LEDs freeze at entry-time state during co-run — verified harmless in real use.

### Performance / UX
- **Performance View knob LEDs show looper state.** In Performance View (locked), each knob LED lights in its track color when that track's looper is on, and goes dark when off. **Knob touch** toggles the looper for that track with a `LOOPER ON / TRACK N` popup. The track menu reflects the change.
- **Unified step-entry velocity rule.** Velocity written to a step now follows one rule.
  - **Drum tracks:** an active drum vel-pad press (while a step is held) wins; else the sticky vel-zone if a vel-pad has been pressed on that track; else **VelIn** if set; else **100**. Drum vel zones always beat VelIn — they are the most direct, intentional control.
  - **Melodic tracks:** **VelIn** (when set 1–127) wins; else a live pad press wins (chord-entry hold-step + press-pad); else **100**.
  - Previously VelIn was ignored at step-write time, drum step-tap defaulted to zone 12 (≈104) even with no vel-pad pressed, and melodic step-tap inherited `lastPadVelocity` from incidental pad activity. Now consistent and predictable.
- **Nudge knob folded onto the Shft knob.** In CLIP, DRUM LANE, and ALL LANES banks, K3 (Nudge) is removed and `Shift + K2 (Shft)` now performs Nudge. The K2 OLED label flips to `Nudg` while Shift is held. Nudge mode is 2× snappier than Clock Shift (sens=4 vs 8). On melodic CLIP, K4–K8 shift down by one (Res → K3, Len → K4, SqFl → K7). On drum lanes / ALL LANES, the middle knobs shift left but K7/K8 (LaneOct/LaneNote on drum CLIP; unused on ALL LANES) stay in place.
- **Loop** and **Capture** buttons now light with a visible dim grey ambient (Loop was invisible at peer-button palette idx 16; Capture was unlit). Loop uses a custom-RGB scratch palette entry; Capture uses DarkGrey. Loop's perf-mode and drum-repeat indicator overrides are preserved.
- **Shift + jog wheel** in Session View now steps the active track 0–7 (was a no-op). Plain jog still scrolls scene rows. Track View behavior unchanged.
- **NoteFX Gate** knob is now 4× snappier: each detent changes the value by ±2% (was 1% per 2 detents). Useful precision preserved for the 0–400% range; other NoteFX knobs unchanged.
- **NoteFX Quantize** knob now changes by ±2% per detent (was ±1%), 2× faster sweep across the 0–100% range.
- **Melodic step-edit Pit knob** snappier: shifts a scale degree every 10 detents (was 16).
- **SEQ ARP / ARP IN `Stps` knob labels spelled out:** `Mut` → `Mute`, `Stp` → `Skip` (Off unchanged). Easier to read at a glance.

### Fixes
- **Shift step-hint overlay drops when a compound modifier is pressed.** Holding Shift normally lights the step buttons as a shortcut-hint grid. When another modifier is pressed *while Shift is held* (Shift+Mute = Solo, Shift+Delete = clear/hard-reset, Shift+Copy = Cut, Shift+Loop = Perf latch toggle), the hint grid is no longer accurate — the step row carries the compound's semantic instead. The hint now disappears the moment the second modifier presses down and reappears if it's released while Shift is still held. Applies to step main LEDs (Track View), icon LEDs (CCs 16-31), and the Session View step-main overlay.
- **Drum-lane step copy now flashes the source step.** Hold Copy + tap a step on a drum-mode track and the source step now blinks white/off the same way it does on melodic tracks. The drum step-LED renderer was returning early before the copy-source overlay; the overlay now applies to both pad modes.
- **Hanging notes during fast or polyphonic live play.** When a pad press and its release landed in the same JS tick (~10.6 ms window), the live-note flush sorted offs before ons unconditionally, so the off processed an inactive note (silent no-op) and the on then activated it with no subsequent off — leaving the note stuck on the Move synth. Drain now emits same-pitch off+on pairs in arrival order while preserving the existing offs-first ordering across different pitches. Affected ROUTE_MOVE most visibly (no downstream chain to mask the hang); ROUTE_SCHWUNG benefits too.
- **Step-entry velocity inconsistency.** Tapping a step button (without holding a pad) now writes a fixed velocity of **100** instead of inheriting `lastPadVelocity` — which was silently being mutated by drum velocity-pad presses, so step taps on melodic tracks would silently shift after touching a drum vel-pad. Affects melodic step-tap (with last-played note) and the multi-toggle path when a primary step is held. Chord-entry (hold step + press pad) is unchanged — pad press velocity still wins.
- **Shift+Step 2/7/9 menu shortcuts** now target by label (`Global` / `Swing Amt` / `Scale`) instead of hardcoded numeric menu indices. The conditional `Edit Slot...` / `Edit Synth...` items were shifting the indices so the shortcuts landed on the wrong items (e.g. Step 7 was hitting `Launch`, Step 9 was hitting `Tap Tempo`). Now stable under future menu reorders.
- MIDI DLY **Lvl** now defaults to 127 on **all drum lanes across all 8 tracks** (was 0 on tracks 1–7 — only track 0 got the JS-side seed). This also makes **Vel Fdbk** audible on those drum lanes: previously, with `Lvl=0` the DSP skipped all delay scheduling, so even non-zero Vel Fdbk produced no effect. Set `Rep ≥ 1` and Vel Fdbk now shapes the repeat-velocity slope as documented. Defaults fixed at the DSP layer (both `drum_pfx_params_init` and `clip_pfx_params_init`); the JS first-run seeding workaround is removed. Saved sets with explicit per-clip/per-lane delay_level keep their stored values; sparse-default sets pick up the new 127 default on next load.
- Panic (Delete+Play when stopped) now sweeps all 16 MIDI channels on every active route, not just the channels that tracks happen to be configured for. ROUTE_SCHWUNG: 128 note-offs × 16 channels; ROUTE_EXTERNAL: CC 120+123 × 16 channels; ROUTE_MOVE: CC 123 × 16 channels (in addition to the existing targeted active-note silencing).

### Persistence
- UI sidecar bumped to v=6: per-track-per-lane Euclidean hit count (`dleu`, 8 × 32 ints) — feeds the DRUM LANE K4 `Eucl` knob. Old v=5 sidecars load with all entries defaulting to 0 (no Euclidean stamps active).
- UI sidecar bumped to v=5: per-track `drumVelZoneArmed` (`dva`) — tracks whether a velocity-pad has been pressed on each drum track, gating the sticky vel-zone fallback for step entry. Old v=4 sidecars load with `dva` defaulted to `false` for all tracks (re-arm by pressing a vel-pad once).
- UI sidecar bumped to v=4: per-track Schwung-slot assignment (`ss`). Old v=3 sidecars upgrade in place; new slots default to unassigned.
